# P2P Radio

校园网 P2P 音频电台。台长在一端播放，听众在另一端实时收听。

## 快速开始

```bash
cd server
npm install
npm start
```

浏览器打开 `http://localhost:3000`。想用 HTTPS（远程主播必需）：

```bash
node generate-cert.js   # 生成自签名证书，只需一次
npm start               # 自动检测证书，启用 HTTPS
```

## 使用方式

页面打开后：

1. 上方显示**活跃电台列表**，点击电台名自动填入输入框
2. 选择角色——**主播**负责播放，**听众**负责收听
3. 点击"加入"

| 角色 | 操作 | 需要权限 |
|---|---|---|
| 主播 | 加入后手动点"麦克风：开"或"系统声音：开" | 麦克风 / 屏幕共享 |
| 听众 | 加入即听，无需额外操作 | 无 |

**主播**：点击"系统声音：关"按钮，在弹出窗口中选择任意窗口或标签页，并勾选"共享音频"。视频画面会被立刻丢弃，仅保留音频。状态栏显示"画面已丢弃"作为确认。

**听众**：无权限弹窗，加入后自动收听。如果电台已有主播，角色选择中的"主播"按钮会被自动锁定，只能以听众身份加入。

**离开**：点击红色"离开"按钮，随时可加入其他电台。

## 架构

```
台长 ──WebRTC──→ 听众 1
  │──WebRTC──→ 听众 2        mesh 拓扑，音频直连不经服务器
  └──WebRTC──→ 听众 N

服务器仅做信令（WebSocket）+ STUN（UDP 3478），
音频流 P2P 直传，不走服务器。
```

- **服务器**：Express + WebSocket 信令 + 内嵌 STUN（无需 Google STUN）
- **客户端**：原生 JS，无框架
- **音频**：Opus 编码，立体声 256kbps，FEC 前向纠错
- **播放**：听众用 `<audio>` 元素；iOS Safari 用 Web Audio API（绕过 autoplay 限制）
- **断线恢复**：WebSocket 自动重连（指数退避），ICE 失败自动重启

## 诊断面板

页面底部折叠区，点 `▶ 诊断面板` 展开。

- **缓冲 (ms)**：当前音频在抖动缓冲中等待的时间。如果频繁接近 0 且伴卡顿，说明 Wi-Fi 抖动严重
- **Loss (%)**：丢包率。FEC 可恢复单包丢失，连续丢包才会卡顿
- **角色标签**：每行标注 `🎤主播` 或 `🎧听众`

## 手动验证

1. 启动服务器
2. 标签页 A：选择"主播"，加入房间 `test`，打开麦克风或系统声音
3. 标签页 B：选择"听众"，加入同一房间
4. 标签页 B 的播放状态区应显示电平律动和声音

## API

| 端点 | 说明 |
|---|---|
| `GET /api/rooms` | 当前活跃电台及其主播/听众人数 |
| `GET /api/access-url` | 服务器局域网地址 |

## 限制

- mesh 拓扑适合 2-10 人。更大规模建议换 SFU（如 mediasoup）
- 自签名证书下浏览器会提示"不安全"，需手动点"继续访问"
- 校园网跨子网需依赖内嵌 STUN；Symmetric NAT 需 TURN 中继

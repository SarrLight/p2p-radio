
## 1. 项目愿景

打造一个极轻量级的校园 P2P 实时音频空间。用户在静态地图上“插眼”开播，校友通过地图方位发现并零延迟加入“一起听”。

## 2. 核心流程 (Independent Mode)

* **台长端 (Host):**
1. 点击地图上的某个坐标 $(x, y)$。
2. 授权浏览器录屏/音频权限 (`getDisplayMedia`)。
3. 系统生成一个随机 6 位 Room ID，并启动 PeerJS 监听。
4. 通过 WebSocket 向后端注册房间信息（坐标、Room ID、Peer ID）。


* **听众端 (Listener):**
1. 进入页面，通过 WebSocket 获取当前所有活跃房间。
2. 在地图上渲染“呼吸灯”图标。
3. 点击图标，通过 PeerJS 与台长建立 P2P 连接，接收并播放音频流。



## 3. 技术栈约束

* **前端:** Vue 3 (Composition API), Vite, Tailwind CSS.
* **通讯:** PeerJS (WebRTC 封装), WebSocket (信令交换)。
* **后端:** Spring Boot (Java), WebSocket Starter.
* **部署环境:** 优先考虑校园局域网环境，利用 P2P 实现高带宽、低延迟传输。

## 4. 功能模块划分

### 4.1 空间沙盘 (Map Module)

* **底图:** 全屏展示一张静态校园手绘/像素地图（后期支持平移缩放）。
* **坐标系:** 使用 **百分比坐标** ($0\% \sim 100\%$) 以适配不同分辨率的屏幕。
* **视觉效果:** 活跃电台在地图上显示为带有半径扩散效果的“呼吸灯”。

### 4.2 音频引擎 (Audio Engine)

* **采集:** 必须勾选“分享系统音频”。
* **传输:** 纯 P2P 架构，音频流不经过 Spring Boot 后端。
* **并发:** MVP 阶段每个房间最大连接数限制在 **15 人**。

### 4.3 社交交互 (Social Lite)

* **实时弹幕:** 使用 PeerJS 的 `DataChannel` 实现房间内匿名聊天。
* **生命周期:** 网页关闭即下线，后端 WebSocket 监听 `onClose` 自动清理地图点位。

## 5. UI/UX 设计准则 (The Vibe)

* **色调:** 全局暗色模式 (`#0a0a0a`)，强调色使用萤火绿 (`#4ade80`) 或深天蓝。
* **质感:** 毛玻璃效果 (Glassmorphism)，浮窗需有半透明感。
* **反馈:** 点击地图、音频连接成功、有新听众加入时，需有微小的视觉反馈（如波纹效果）。
